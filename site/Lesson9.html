Step 2. Naming Conventions

In the example above we created a class that registered two fonts. Take note of the [Embed()] tag. For this example we chose to use the property fontFamily. This is much preferred over the property fontName. fontFamily allows us to assign multiple fonts under one name, in this case "Arial" and distinguish them by applying font properties such as weight or italics. By using fontFamily we can create a style sheet that uses a fontFamily of Arial. We can also apply a font-weight: bold; property to use the now embedded bold type. It is recommended that you compile the above ActionScript 3 into a swf that matches the font name (in this case "Arial.swf" is preferred). This will help avoid confusion when using the font.

First we load the swf "Arial.swf". By loading the font it is already registered since we called the Font.registerFont() in the ActionScript 3. Once the org.fxml.loaders.BulkLoader completes the loading, we can simply use the font by creating a flash.text.TextField object, setting the embedFonts equal to true and then setting the font property to "Arial" on the defaultTextFormat object on the flash.text.TextField!
