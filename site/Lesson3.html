Lesson 3. The Application.

In this example you will learn about some additional features of org.fxml.Application class

The application class (org.fxml.Application), is the document class of the application. It is the entry point and is the top level flash.display.Sprite. Because org.fxml.Application extends flash.display.Sprite, any public properties or methods of flash.display.Sprite are available to you.

For full documentation please reference the ASDocs.

We will cover the following topics in this lesson:

  • this
  • addChild()
  • version
  • pause()
  • resume()
  • Nested Application

this

In ActionScript the keyword "this" is defined as the following: "A reference to a method's containing object. When a script executes, the this keyword references the object that contains the script. Inside a method body, the this keyword references the class instance that contains the called method."

In FXML the this keyword is used as a reference to the org.fxml.Application object. This allows access to the properties and methods on the org.fxml.Application object at any depth. This is important, for without the use of the keyword this, the scope will default to the encompassing object. Properties and methods set between an object in FXML are referenced simliar to using the with keyword.

We will see more examples of this in the next two lessons.

pause()

Calling the pause method will pause the FXML parser. Since FXML does a single-pass, calling pause will halt the application until the resume method is invoked.

resume()

Calling the resume method resumes the FXML parser.
Note: you can not directly call the resume method. It must be invoked as a handler from an event listener.
